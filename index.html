Spring Config

简单来说就是一个配置中心的服务作为服务端，其他服务作为客户端获取配置信息。

除了有服务端，另外需要一个仓库来存放这些配置文件，可以用github，svn，数据库，本地等存储。这里用github举例。

创建config-repo

目录大概这样，link-development，link代表项目名，development代表版本，这里是开发版本，最终客户端获取link的配置时，会把application，link，再加上对应profile的配置合并到一起，如果有重复配置以application为主。

applicaiton.yml

boostrap.yml

yml中—的作用是用来隔离不同profile的配置

启动之后可以在浏览器直接访问。Spring Cloud Config 有它的一套访问规则，我们通过这套规则在浏览器上直接访问就可以。

  • {application} 就是应用名称，对应到配置文件上来，就是配置文件的名称部分，例如上面的link,linflow。
  • {profile} 就是配置文件的版本，对应到配置文件上来就是以 link-{profile}.yml 加以区分，例如link-development.yml、link-integration.yml、link-test.yml等。
  • {label} 表示 git 分支，默认是 master 分支，如果项目是以分支做区分也是可以的，那就可以通过不同的 label 来控制访问不同的配置文件。

Mockito

为什么使用

在进行单元测试的时候，当前测试对象依赖很多其他的类，你需要手动创建出这些依赖对象，这个时候你可以用Mockito来模拟那些依赖的类。

所谓mock就是创建一个类的虚假对象，用来替换真实对象来达到两个目的:

  • 验证这个对象的某个方法的调用情况，调用了多少次，参数是什么
  • 指定这个对象的某些方法的行为，返回特定的值，或者执行特定的动作

如何使用

一般配合junit使用，引入junit和mockito依赖

验证某些行为

验证mockList有没有执行过add(1)和clear()方法,一旦mock对象被创建了，mock对象会记住所有的交互。

打桩

mock函数默认返回的是null，一个空的集合或者一个被对象类型包装的内置类型，例如0、false对应的对象类型为Integer、Boolean；你可以指定调用时返回固定的值或者抛出异常等。

参数匹配器
验证函数的确切、最少、从未调用次数

微信第三方平台对接

微信第三方平台授权流程梳理

  • component_verify_ticket ：验证票据，在第三方平台创建审核通过后，微信服务器会向其 ”授权事件接收URL” 每隔 10 分钟以 POST 的方式推送 component_verify_ticket。可用于获取component_access_token

  • component_access_token：第三放平台的access_token,可用于调用第三方平台的api,需要跟authorizer_access_token区别开

  • pre_auth_code：预授权码，通过component_access_token获取，可用于拼接公众号授权给第三方平台的扫码授权页面

  • authorization_code(auth_code)：在公众号同意授权后返回给第三方平台,可用于和component_access_token获取authorizer_access_token和authorizer_refresh_token

  • authorizer_access_token：用于代替授权的公众号去调公众号相关接口

  • authorizer_refresh_token ：用于刷新authorizer_access_token

首先是等待微信推送component_verify_ticket，用ticket获取component_access_token，用access_token获取pre_auth_code

然后拼接授权页面，授权完成后拿到auth_code。用authorization_code和component_access_token获取authorizer_access_token和

authorizer_refresh_token

然后用authorizer_access_token就可以调用公众号相关接口。
