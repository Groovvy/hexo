Spring Config

简单来说就是一个配置中心的服务作为服务端，其他服务作为客户端获取配置信息。

除了有服务端，另外需要一个仓库来存放这些配置文件，可以用github，svn，数据库，本地等存储。这里用github举例。

创建config-repo

目录大概这样，user-development，user代表项目名，development代表版本，这里是开发版本，最终客户端获取user的配置时，会把application，link，再加上对应profile的配置合并到一起，如果有重复配置以application为主。

创建配置中心服务端

applicaiton.yml

boostrap.yml

yml中—的作用是用来隔离不同profile的配置

启动之后可以在浏览器直接访问。Spring Cloud Config 有它的一套访问规则，我们通过这套规则在浏览器上直接访问就可以。

  • {application} 就是应用名称，对应到配置文件上来，就是配置文件的名称部分，例如上面的user。
  • {profile} 就是配置文件的版本，对应到配置文件上来就是以 user-{profile}.yml 加以区分，例如user-development.yml、link-production.yml等。
  • {label} 表示 git 分支，默认是 main分支，如果项目是以分支做区分也是可以的，那就可以通过不同的 label 来控制访问不同的配置文件。

创建配置中心客户端，使用配置

微信第三方平台对接

发表于 2021-04-07 更新于 2021-04-15

微信第三方平台授权流程梳理

  • component_verify_ticket ：验证票据，在第三方平台创建审核通过后，微信服务器会向其 ”授权事件接收URL” 每隔 10 分钟以 POST 的方式推送 component_verify_ticket。可用于获取component_access_token

  • component_access_token：第三放平台的access_token,可用于调用第三方平台的api,需要跟authorizer_access_token区别开

  • pre_auth_code：预授权码，通过component_access_token获取，可用于拼接公众号授权给第三方平台的扫码授权页面

  • authorization_code(auth_code)：在公众号同意授权后返回给第三方平台,可用于和component_access_token获取authorizer_access_token和authorizer_refresh_token

  • authorizer_access_token：用于代替授权的公众号去调公众号相关接口

  • authorizer_refresh_token ：用于刷新authorizer_access_token

Mockito

发表于 2021-03-16 更新于 2021-04-15

为什么使用

在进行单元测试的时候，当前测试对象依赖很多其他的类，你需要手动创建出这些依赖对象，这个时候你可以用Mockito来模拟那些依赖的类。

所谓mock就是创建一个类的虚假对象，用来替换真实对象来达到两个目的:

  • 验证这个对象的某个方法的调用情况，调用了多少次，参数是什么
  • 指定这个对象的某些方法的行为，返回特定的值，或者执行特定的动作
