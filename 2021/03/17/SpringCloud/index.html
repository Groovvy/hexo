Spring Config

简单来说就是一个配置中心的服务作为服务端，其他服务作为客户端获取配置信息。

除了有服务端，另外需要一个仓库来存放这些配置文件，可以用github，svn，数据库，本地等存储。这里用github举例。

创建config-repo

目录大概这样，link-development，link代表项目名，development代表版本，这里是开发版本，最终客户端获取link的配置时，会把application，link，再加上对应profile的配置合并到一起，如果有重复配置以application为主。

创建配置中心服务端

applicaiton.yml

boostrap.yml

yml中—的作用是用来隔离不同profile的配置

启动之后可以在浏览器直接访问。Spring Cloud Config 有它的一套访问规则，我们通过这套规则在浏览器上直接访问就可以。

  • {application} 就是应用名称，对应到配置文件上来，就是配置文件的名称部分，例如上面的link,linflow。
  • {profile} 就是配置文件的版本，对应到配置文件上来就是以 link-{profile}.yml 加以区分，例如link-development.yml、link-integration.yml、link-test.yml等。
  • {label} 表示 git 分支，默认是 master 分支，如果项目是以分支做区分也是可以的，那就可以通过不同的 label 来控制访问不同的配置文件。

创建配置中心客户端，使用配置

服务发现与注册 Eureka

服务端

引入相关依赖，在启动类上添加注解@EnableEurekaServer，将此项目作为Euraka注册中心。

bootstrap.yml

eureka.client.register-with-eureka: false 禁止自己当做服务注册

eureka.client.fetch-registry: false 屏蔽注册信息

客户端

引入相关依赖，在启动类添加@EnableDiscoveryClient或者@EnableEurekaClient，这两个的区别是后者只能使用Eureka作为注册中心，前者可以使用其他的。
